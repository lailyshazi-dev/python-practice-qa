# День 06: GitHub и push

## Что я сделал

- Создал репозиторий на GitHub.
- Подключил локальный проект к GitHub через `remote`.
- Отправил ветку `master` на GitHub через `git push`.
- Проверил, что коммиты появились в удаленном репозитории.

## Что понял

- Git хранит историю проекта локально.
- GitHub хранит копию репозитория в интернете.
- `origin` - это имя удаленного репозитория.
- `git push` отправляет локальные коммиты на GitHub.
- После первого `push -u` можно использовать просто `git push`.

## Новые слова

- `remote` - удаленный репозиторий.
- `origin` - стандартное имя удаленного репозитория.
- `push` - отправка локальных коммитов на GitHub.
- `upstream` - связь локальной ветки с удаленной веткой.
