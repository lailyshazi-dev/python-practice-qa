# День 05: ветки и merge

- Добавил функцию `average`.
- Добавил параметризованные тесты для `average`.
- Сделал коммит в feature-ветке.
- Переключился обратно в `master`.
- Влил изменения через `merge`.
- Удалил feature-ветку после merge.

## Что понял

- Ветка - это отдельная линия работы над задачей.
- `HEAD` показывает текущий коммит и текущую ветку.
- `git switch -c` создает новую ветку и сразу переходит в нее.
- `git merge` переносит изменения из одной ветки в другую.
- После успешного merge feature-ветку можно удалить.

## Новые слова

- `branch` - ветка.
- `merge` - слияние изменений.
- `fast-forward` - простой merge, при котором Git двигает указатель ветки вперед.
- `feature branch` - ветка под отдельную задачу.
