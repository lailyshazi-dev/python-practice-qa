# День 03: Git

## Главная идея

Git хранит историю проекта. Коммит - это сохраненная точка состояния проекта.

## Базовый цикл

```powershell
git status
git add .
git commit -m "Initial project setup"
```

## Что означают команды

- `git status` - показывает состояние файлов.
- `git add` - подготавливает файлы к коммиту.
- `git commit` - сохраняет подготовленные изменения в историю.
- `git log` - показывает историю коммитов.

## Важные зоны Git

- Working tree - файлы, которые мы редактируем.
- Staging area - файлы, подготовленные к коммиту.
- Repository - сохраненная история коммитов.

## Правило дня

Перед коммитом всегда запускаем тесты.
